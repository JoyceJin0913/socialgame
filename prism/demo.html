PRISM · 世界切片多样性方法

把一段剧情
变成N 个世界。

一种为互动文游设计的方法论。
让作者不必为每个分支写一遍，让玩家在不同世界里看到不同戏。

↓ SCROLL TO BEGIN
STEP 01 · 问题

大多数互动文游的"分支"
其实只是装饰。

你点 A，三句话之后回主线。你点 B，三句话之后也回主线。
选项写了三个，玩家走了三遍发现都一样——这就是"假分支"。

传统分支树

3 个选项 = 3 段不同文本

每个选项作者写一段台词，互相之间不影响，三句话后剧情合流。

节点："傅云夕凯旋归来" ├─ 选项 A："我等你回来" → 合流 ├─ 选项 B："你必须回来" → 合流 └─ 选项 C："……好" → 合流
↓
所有人都看到同样的下一幕
Prism 折射

3 个选项 × N 个世界 = 涌现

选项只是一份候选清单。具体哪几个出现、代价多大，由"当前世界"决定。

节点："傅云夕凯旋归来" ├─ 在正堂当众 → 仅 3 项可见 ├─ 在偏院私下 → 多 2 项浮出 └─ 在暗夜独处 → 不同的代价倍率
↓
每个玩家看到不同的戏
STEP 02 · 洞察

节点不是写好的内容，
是一个生成器函数。

我们换一种思考方式。作者不再写"在 X 场景下，玩家有 3 个选项"。
作者写：这个节点必须发生什么戏剧功能，加上 一份候选选项池。
具体长什么样、有哪些选项、代价多少——由当前世界状态在运行时算出来。

作者写：晶体
×
当前世界状态
=
玩家看到的这一幕
晶体 是不变的（必须发生什么）；世界状态 是变化的（在哪、谁在、几点、什么舆论）。
乘起来就是玩家此刻看到的戏。
STEP 03 · 晶体

先讲晶体（Crystal）。
它是节点的"骨架"。

晶体只描述戏剧功能，不描述具体场景。它是节点中不变的部分。
我们用一个真实例子。

CRYSTAL · 案例

"男主与女主的告别 / 离心时刻"

必须发生
· 男主主动制造"距离"（出征 / 冷淡 / 休书）
· 女主必须做出第一次回应
· 男主真实意图（隐疾 / 保护）此刻不能直接说出
必须不发生
· 男主当面坦白寒毒
· 女主与男主直接动手
戏剧功能
让玩家亲历"明明在乎，却必须放手"的张力

这个晶体可以发生在雪夜书房（一年之约）、金銮殿外（公主当庭）、深夜独对（一纸休书）——三种场景都满足以上"必须"。
晶体不绑定场景。绑定场景的，是接下来要讲的五轴。

STEP 04 · 五轴

世界被分成五条独立的轴。
每个轴管一件事。

五轴是正交的——你可以单独调任何一个轴，不会破坏剧情的"必须发生"。
下面一个一个讲，每个轴都用同一个晶体演示只调这一个轴会发生什么。

S
SCENE
场景舞台

事件发生在哪里。

同一段戏，在"大庭广众"和"密室单独"下，玩家能用的道具、敢说的话、付出的代价完全不同。

同一晶体，只调 S 轴
S = 雪夜书房退路：可走可留 · 旁观者：无 · 代价：每句话直入对方心
S = 金銮殿外退路：无 · 旁观者：满朝文武 · 代价：每一句记入起居注
H
HIDDEN
隐藏信息

玩家知道多少真相。

同一段对峙，玩家"完全不知道男主装的"和"已经隐约察觉男主装的"，能做的选择完全不同。这是 Prism 最强的多样性引擎。

同一晶体，只调 H 轴
H · 男主隐疾 = 未知玩家以为他变了心，会出现"撕休书 / 质问"等情绪化选项
H · 男主隐疾 = 暗示多出"借故凑近闻药味 / 试他袖中药瓶"等理性选项 NEW
N
NPC
在场人物

谁在、谁缺席、谁在暗处看。

"半在场"（戴面具、躲屏风后）尤其关键——它让同一场戏可触发的回调完全改变，却不破坏戏剧功能。

同一晶体，只调 N 轴
N = {男主, 心腹}私下相对，情绪化选项可触发
N = {男主, 西戎公主, 朝臣}出现"借舆论反将公主一军"等公开博弈选项 NEW
T
TIME
时间压力

距离下一个不可逆事件还有多久。

时间不直接改变选项内容，但决定哪些选项还来得及做。当时间逼近，"找师父商量"会消失，"立刻硬刚"会浮出。

同一晶体，只调 T 轴
T = 一夜可选项：试探、追问、布局，时间足够慢慢谈
T = 半炷香长线选项消失，浮出"立刻拦下他 / 当场表态"等紧急路径 NEW
A
AMBIENT
环境噪声

世界对玩家选择的放大或反弹。

流言风向、天气、市井情绪——它们不改变选项本身，但是选项代价的乘数。雪夜里硬刚的反响是平日的两倍。

同一晶体，只调 A 轴
A · 舆论 = 私密 / 雪夜将至所有选择的代价 ×1.0，无人评判
A · 舆论 = 满朝传遍 / 天气 = 大雪所有代价 ×2.0，硬话的回响倍数放大
STEP 05 · 组合

把五轴组合起来，
就得到一个"世界切片"。

五轴中的每个轴有 2–3 个选择，组合起来理论上能产生几十种切片。
我们手动挑出 3 个最有戏剧差异的，分别命名为 α、β、γ。

命名约定： α、β、γ 只是这三个切片的代号（像变量名 a、b、c），
它们对应的是不同的五轴配置，不是不同的剧情节点。
同一个晶体，三种切片，三个世界。

STEP 07 · 协作

Prism 不接管数值，
它和数值系统联调。

Prism 只回答一个问题："在这个世界切片下，玩家此刻能看到哪些选项？"
"每个选项加多少分、扣多少声望、累积到什么阈值"——这是数值系统的活。
两者通过一个叫做 numerics_hook 的接口位对接。

◆ 与数值系统（Brewed）的接口契约

Prism 在每个选项里只声明一件事：这个选项发生时，向数值系统抛出哪个语义事件。

例如选项"撕休书"会抛出 event: "trust_collapse"；数值系统决定这个事件值多少 courage、扣多少 trust_fuyunxi。

这种分工的好处：

· 改剧情：只动 Prism（加 slice、改 require）

· 改数值平衡：只动数值系统（调 event → 数值的映射表）

· 双方互不踩脚，可独立迭代
