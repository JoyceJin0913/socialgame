#### Rule 2: 多用户互动协议（Multi-User Interaction Protocol）

```
【聊天频道系统】
📢 公开频道：所有玩家可见，用于公开发言、指控、辩论
💬 私聊频道：1对1私密对话，可用于结盟/欺骗/交易
🎭 角色扮演频道：纯RP内容，增强沉浸感
📋 投票频道：用于集体决策、放逐投票
🎁 交易频道：物品/秘密/技能的交易市场

【实时互动处理流程】
1. 玩家A发起行动 → 
2. AI判断是否涉及其他玩家 →
   ├── 是 → 暂停处理，通知相关玩家响应（30秒窗口）
   └── 否 → 直接结算结果
3. 收集所有相关输入后统一结算
4. 广播结果（根据秘密级别决定可见范围）
5. 更新全局状态（关系网、势力图、任务进度）

【并发冲突解决】
- 当多个玩家同时对同一目标采取互斥行动时：
  - 按「速度属性」判定先后顺序
  - 先执行者的结果会影响后续行动的成功率
  - 特殊技能（如「扭转乾坤」）可插队
```

#### Rule 3: 分支剧情引擎（Branching Narrative Engine）

```
【分支类型定义】

TYPE A: 短期分支（微蝴蝶效应）
├── 影响范围：当前场景 + 下1个场景
├── 回归点：最多2个场景后汇回主线
├── 示例：选择是否参加宴会 → 影响遇到谁 → 不影响主线进程
└── 生成规则：AI自由发挥，无需人工预设

TYPE B: 中期分支（情节分歧点）
├── 影响范围：整章剧情走向
├── 回归点：章节末尾强制汇合
├── 示例：选择救/不救某角色 → 不同的发展路线 → 关键节点汇合
└── 生成规则：需检查预设条件，允许有限偏离

TYPE C: 长期分支（世界线变动）
├── 影响范围：后续全篇
├── 回归点：无（或多结局之一）
├── 示例：主角黑化 / 重要角色死亡 / 阵营重组
└── 生成规则：严格校验，需满足阈值条件
    - 累计决策点≥5个同方向选择
    - 关键角色死亡/叛变
    - 使用终极技能改变现实

【分支合并机制】
当检测到以下情况时，触发强制合并：
1. 到达预设的「锚点场景」（如大婚/决战）
2. 游戏进入下一阶段（Phase Transition）
3. 故事逻辑出现无法修复的断裂

【合并策略】
- 「幽灵选择」：未被大多数玩家选择的路线以NPC/回忆形式存在
- 「世界记忆」：即使合并，之前的分支选择仍影响角色状态和关系
```

### 4. 结局判定算法（Ending Determination Algorithm）

```python
def calculate_ending(game_state):
    """
    根据游戏状态计算最终结局
    """
    
    # 核心指标收集
    revenge_score = game_state.zhuang_hanyan.revenge_progress  # 复仇进度 0-100
    romance_score = game_state.fu_yunxi.zhuang_hanyan_affinity  # 感情线 0-100  
    villain_fate = game_state.antagonist_group.defeat_level     # 反派败亡程度 0-100
    character_deaths = len(game_state.dead_characters)          # 死亡人数
    player_choices = extract_key_decisions(game_state.decision_log)  # 关键决策
    
    # 结局权重计算
    endings = {}
    
    # True Ending (Good End)
    endings["revenge_he"] = (
        revenge_score >= 80 and 
        romance_score >= 70 and 
        villain_fate >= 90 and 
        character_deaths <= 2
    ) * 0.35
    
    # Redemption Ending (Normal End)
    endings["redemption_be"] = (
        revenge_score >= 50 and 
        villain_fate >= 60 and 
        "forgive" in player_choices
    ) * 0.25
    
    # Tragedy Ending (Bad End)
    endings["tragedy_te"] = (
        revenge_score < 30 or 
        zhuang_hanyan_dead or 
        romance_score < 20
    ) * 0.2
    
    # Dark Ending (Special End)
    endings["dark_de"] = (
        "betray_allies" in player_choices and 
        "kill_innocent" in player_choices and
        revenge_score >= 90
    ) * 0.15
    
    # Hidden Endings (特殊条件解锁)
    if unlock_condition("all_secrets_found"):
        endings["truth_end"] = 0.05
        
    if unlock_condition("everyone_survived"):
        endings["peace_end"] = 0.05
        
    if unlock_condition("harem_route"):
        endings["harem_end"] = 0.03  # 彩蛋结局
    
    # 返回最高分结局
    return max(endings.items(), key=lambda x: x[1])
```

### 5. 实时状态同步协议（Real-time State Sync）

```
【状态广播格式】
{
  "timestamp": "ISO8601",
  "event_type": "state_update | action_result | dialogue | system_message",
  "visibility": "public | faction | private | gm_only",
  "payload": {

【客户端显示规则】
- 公开信息：所有玩家界面实时更新
- 阵营信息：仅同阵营玩家可见
- 私密信息：仅目标玩家可见
- GM信息：仅主持人/DM可见

【冲突解决】
当两个客户端状态不一致时：
1. 以服务端状态为准
2. 客户端提交冲突报告
3. GM/仲裁系统裁决
4. 广播最终状态
```
